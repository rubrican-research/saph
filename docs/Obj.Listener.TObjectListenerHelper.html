function objectAlive: boolean;

Returns total number of all signals that are active at the present moment. This is not the count of active signals of this object. Is this object currenly alive ? hack to handle cases where the memory address will be accessed but we don't know if the object has been freed.

procedure stopListening;

You should call this in the destructor of your objects that are listening to signals. This is for listener objects, not signalling. It removes all methods that were added as listeners across all objects in the library.

procedure beforeDestruction;

Implicitly hiding beforeDestruction so that we can hack into object destruction and remove listeners
